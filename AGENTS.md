# AGENTS.md

## UI Work

When implementing, redesigning, or refactoring UI in this repository, read `docs/ui-direction.md` first and use it as the product and UX baseline.

Apply that document to:

- Screen layout and information hierarchy
- Visual styling and design-token decisions
- Component usage and interaction patterns
- Accessibility, motion, loading, and feedback behavior
- Copy tone for UI labels, descriptions, and status text

If a requested UI change conflicts with the direction in that document, call out the conflict and either:

1. ask for confirmation before breaking the direction, or
2. note the override clearly in the final response if the user explicitly requested it.
